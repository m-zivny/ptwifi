# ptwifi

## Overview
<hr>
ptwifi is a command-line tool for analyzing wireless IEEE 802.11 networks with a primary focus on detecting and analyzing wireless Access Points. 

ptwifi combines Python-based processing with external wireless auditing tools to collect and analyze information about Access Points in real time. 

The main objectives of the tool are:

- detection and identification of nearby WiFi Access Points
- analysis of Access Point parameters such as channel, signal strength, and security configuration
- observation of client stations associated with detected Access Points
- performing controlled wireless testing against Access Points (e.g., deauthentication testing)
- exporting collected Access Point data in structured formats for further analysis
- integrating different wireless analysis into one workflow
- simplifying wireless network testing workflows

The tool is designed primarily for educational and research purposes in wireless networking and wireless network security analysis.
## Features
<hr>

- Passive scanning of WiFi Access Points using `airodump-ng`.
- Active scanning of wireless networks using IEEE 802.11 probe requests.
- Real-time display of preliminary scan results.
- Identification of Access Point parameters such as SSID, BSSID, channel, signal strength, and security configuration.
- Detection of client stations communicating with Access Points.
- Approximate distance estimation based on received signal strength.
- Vendor identification using MAC address OUI lookup.
- Deauthentication testing against specified Access Points.
- Structured export of collected scan results to JSON format.
- Modular architecture allowing easy extension with additional wireless testing modules.

## Architecture
<hr>

### Environment


The tool is intended to be used on **Kali Linux** with a wireless interface
that supports **monitor mode**.

### Project Structure

- **main.py** – entry point of the application

- **helpers/**
  - **interface_setup.py** – wireless interface management
  - **helper_functions.py** – shared utility functions
  - **classes.py** – core data structures and constants
  - **manuf.csv** – OUI database for vendor identification

- **modules/**
  - **passive_scan.py** – passive WiFi scanning using airodump-ng
  - **active_scan.py** – active scanning using probe requests
  - **deauth_attack.py** – deauthentication testing module

- **output/** – directory where scan results are stored

- **requirements.txt** – Python dependencies

- **README.md** – project documentation

### Modules

#### main.py
Main entry point of the ptwifi tool.  
Handles command-line argument parsing, validates runtime conditions, prepares the wireless interface, and executes the selected module.

##### interface_setup.py
Provides functions for querying and configuring wireless interfaces.  
Handles monitor mode switching, channel configuration, and validation of interface capabilities.

##### helper_functions.py
Collection of utility functions used across the project.  
Includes formatting utilities, JSON export, signal strength processing, distance estimation, and vendor lookup.

##### classes.py
Defines core data structures and constants used by the tool.  
Includes Access Point and Station classes as well as enumerations and terminal formatting utilities.

##### passive_scan.py
Implements passive WiFi scanning using the external tool `airodump-ng`.  
Parses CSV output periodically generated by airodump-ng and builds internal representations of detected Access Points and client stations.

##### active_scan.py
Implements active network discovery using IEEE 802.11 probe requests.  
Transmits probe frames across selected channels and collects probe responses from nearby Access Points.

##### deauth_attack.py
Implements a deauthentication testing module.  
Constructs IEEE 802.11 deauthentication frames and transmits them toward a specified Access Point or client station.

### External Tools
<hr>

The tool integrates several external wireless auditing utilities used for capturing and analyzing WiFi traffic.

- **Aircrack-ng (airodump-ng)** – licensed under the GNU General Public License v2 (GPLv2)  
  https://www.aircrack-ng.org/
- **iw** – used for querying wireless interface capabilities and configuration.
- **ip** – used for bringing network interfaces up or down.
- **nmcli** – used for managing NetworkManager-controlled wireless interfaces.
- **systemctl** – used to restart the wpa_supplicant service when restoring managed mode.

## Usage
<hr>

### Listing Wireless Interfaces
`sudo python3 main.py show ifaces`
### Passive Scan
`sudo python3 main.py wlan0 passive`
### Active Scan
`sudo python3 main.py wlan0 active`
### Deauthentication Test
`sudo python3 main.py wlan0 deauth -b 00:11:22:33:44:55`
    

## Limitations
<hr>

- The tool currently operates primarily on the 2.4 GHz band.
- The number of implemented testing modules is currently relatively small, focusing mainly on scanning and basic deauthentication testing.
- The tool primarily collects and presents data, but the interpretation and evaluation of results are left to the tester.
